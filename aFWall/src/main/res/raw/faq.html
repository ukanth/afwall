Q: Is there a more detailed Frequently Asked Questions (FAQ) online?
A: Yes, there is a detailed online FAQ page on GitHub.

Q: Why is the YouTube app unable to connect?
A: Ensure 'Media Server' in the app list is checked. Some audio and video apps need this to work properly.

Q: Why do I see data traffic for an app that is blocked by AFWall+?
A: It's possibly false data traffic. AFWall+ uses IPtables which is at the kernel level (low-level).
Data use shown by Android is calculated before it gets dropped by the firewall.

Q: Why are downloads not working with stock browsers or the Google Play Store?
A: Allow "Downloads" from the app list.

Q: What's the difference between the two modes?
A: Allow Mode is for users who know exactly which apps they want to be allow to connect to the internet (including system apps).
Block Mode is for users who only know which apps they want to block from connecting to the internet. Block Mode is more suited for beginners.

Q: Why does my entire network seem to be blocked after I enabled the experimental 'Fix Startup Data Leak' option?
A: Please uncheck that option (this will remove the init.d file automatically).
It is under Preferences - Experimental preferences - 'Fix startup data leak'.
In case you uninstalled AFWall+ and copied the script manually, ensure it is removed: the file is called 'afwallstart'.
