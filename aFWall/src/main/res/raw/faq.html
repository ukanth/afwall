Q: Is there a Frequently Asked Questions (FAQ) page?
A: Yes, there is a detailed FAQ page on GitHub.

Q: Why is the YouTube app unable to connect?
A: Ensure 'Media Server' from applications list was checked. Some audio and video apps need this to work properly.

Q: I see data traffic consumed by application 'xyz' which is blocked by AFWall+, why?
A: It's possibly false data traffic. AFWall+ uses iptables, which is at kernel level (low-level).
Usage data shown by Android is calculated before it gets dropped by the firewall.

Q: Downloads are not working with stock Browsers or the Google Play Store
A: Allow 'Downloads' from the application(s) list.

Q: What's the difference between white- or blacklist?
A: Whitelist is for advance users where they know which app's can connect to internet including system application(s).
Blacklist is for beginners where they want to block only certain app's connecting to the internet.
It depends on person to person which mode fits best.

Q: My entire network seems to be blocked after I enabled the experimental 'Fix Startup Data Leak' option.
A: Please uncheck the option (this will remove the init.d file automatically).
It's under Preferences - Experimental Preferences - 'Fix startup Data Leak'.
If you uninstalled AFWall+ and copied the script manually, ensure it's removed. The file is called 'afwallstart'.

Q: What are all the icons on the top?
A: Icons for the following may be visible (depends on the options you enable):

* Local Area Network (LAN) - LAN data connections
* WiFi - Wifi data connections
* Mobile - 2G/3G/4G data connections
* Roaming - mobile data connections outside of the standard service area
* Virtual Private Network (VPN) - data connections
* 'Revert' and 'Clear' selection
